# Plan 04 Summary: Custom Domains

## Phase
2.1-link-shortener

## Plan
02.1-04-PLAN.md

## Objective
Add custom domain support so users can use their own domains for short links, with DNS TXT verification for ownership proof.

## Tasks Completed

### Task 1: Schema + Error Codes
- Added `customDomains` table to `src/db/schema.ts` with columns: id, userId, domain (unique), verificationToken, isVerified, isActive, createdAt, updatedAt
- Added indexes on userId and domain
- Added 4 new error codes to `src/lib/api-response.ts`: DOMAIN_NOT_FOUND, DOMAIN_ALREADY_EXISTS, DOMAIN_NOT_VERIFIED, DOMAIN_VERIFICATION_FAILED

### Task 2: Domain Management API
- Created `src/app/api/domains/route.ts` with GET (list) and POST (create with verification token)
- Created `src/app/api/domains/[id]/route.ts` with GET, PUT (toggle isActive), DELETE
- Created `src/app/api/domains/[id]/verify/route.ts` with DNS TXT verification
- DNS verification uses Node.js `dns.resolveTxt` first, then Cloudflare DoH fallback
- All endpoints enforce ownership via `getOwnedDomain()` helper

### Task 3: Domain Management UI + Routing
- Created `src/components/domain-form.tsx`: domain input, creation, verification instructions with copy button
- Created `src/components/domain-card.tsx`: display domain, status badge, verify button, active toggle, delete
- Created `src/app/[locale]/(dashboard)/settings/domains/page.tsx`: full domain management page
- Updated `src/app/[locale]/(dashboard)/settings/page.tsx`: added Custom Domains card with navigation
- Updated `src/middleware.ts`: async middleware that detects custom domains and skips locale/auth routing
- Updated `src/app/[slug]/route.ts`: `resolveDomain()` helper to distinguish app domain vs custom domain
- Updated `src/app/api/links/route.ts`: POST accepts optional `domain` field, validates ownership
- Updated `src/lib/links.ts`: `isSlugAvailable()` now checks per-domain uniqueness
- Added i18n `domains` namespace to pt-BR, en, es messages files

## Decisions
- DNS verification uses Node.js `dns.resolveTxt` with Cloudflare DoH fallback for environments where native DNS may not work
- Custom domain links skip locale prefix routing entirely
- Slug uniqueness is per-domain (same slug can exist on different domains)
- Only verified and active custom domains can be used for link creation
