# Phase 2.1 Plan 01: Database Schema & Core API Summary

## What Was Built

1. **shortLinks table** — Stores shortened URLs with slug, original URL, optional password, expiration, and click count. Supports both authenticated and anonymous link creation.
2. **linkClicks table** — Records every redirect visit with IP, user agent (parsed into device/browser/OS), referrer, country/city/region placeholders, and timestamp.
3. **POST /api/links** — Creates short links with optional custom slug, password, and expiration. Validates input with Zod, hashes passwords with bcryptjs, applies rate limiting.
4. **GET /[slug]** — Redirects to original URL with 302. Handles inactive, expired, and password-protected links. Records clicks on every visit.
5. **Slug utilities** (`src/lib/links.ts`) — Random slug generation, reserved word blocking, custom slug validation, uniqueness checking.
6. **Click tracker** (`src/lib/click-tracker.ts`) — Parses user agent strings and records clicks with atomic clickCount increment.
