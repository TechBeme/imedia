  <action>
    Create src/app/api/domains/route.ts:
    - GET: list all customDomains for authenticated user, order by createdAt desc
    - POST: create new domain
      - Body: { domain: string } — validate with Zod (must be valid domain format)
      - Generate verificationToken: crypto.randomBytes(32).toString('hex')
      - Insert into customDomains
      - Return success with domain data including verification instructions

    Create src/app/api/domains/[id]/route.ts:
    - GET: fetch single domain by id (ownership check)
    - DELETE: delete domain by id (ownership check)
    - PUT: update domain (toggle isActive)

    Create src/app/api/domains/[id]/verify/route.ts:
    - POST: verify domain ownership
      - Fetch domain from database
      - Use Node.js dns module: dns.resolveTxt(domain, callback)
      - Look for TXT record containing `somedia-verify=${verificationToken}`
      - If found: update isVerified = true, return success
      - If not found: return error("DOMAIN_VERIFICATION_FAILED", "TXT record not found", 400)
      - Handle DNS errors gracefully

    Note: The dns.resolveTxt may not work in all Vercel environments. Add a fallback:
    - Use a public DNS-over-HTTPS API (e.g., Cloudflare's 1.1.1.1) as fallback
    - fetch(`https://cloudflare-dns.com/dns-query?name=${domain}&type=TXT`, { headers: { Accept: 'application/dns-json' } })
    - Parse response for TXT records containing the verification token
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Domain API | Authenticated users only |
| DNS -> Verification | External DNS system provides proof of ownership |
| Custom Domain -> Redirect | Public access to short links on custom domains |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2.1-15 | Spoofing | Domain verification | mitigate | Require DNS TXT record with unique token; token is cryptographically random |
| T-2.1-16 | Elevation of Privilege | Domain hijacking | mitigate | Only verified domains can serve links; verification requires DNS control |
| T-2.1-17 | Information Disclosure | Domain list | mitigate | GET /api/domains only returns domains for authenticated user |
| T-2.1-18 | Denial of Service | DNS verification | mitigate | Cache verification results; limit verification attempts per domain |
</threat_model>
