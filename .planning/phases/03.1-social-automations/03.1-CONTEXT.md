# Phase 3.1: Social Automations — Context

**Gathered:** 2026-05-11
**Status:** Ready for planning
**Source:** User directive (direct conversation)

<domain>
## Phase Boundary

This phase delivers the first automation type in a ManyChat-like social media automation system:
**Comment-triggered automations** that respond to specific keywords in post comments.

The system must be architected as a multi-platform automation engine from day one, even though the first automation type targets Instagram comments. The data model and execution engine must support future automation types (DM triggers, story replies, mentions, etc.) and future platforms (TikTok, YouTube, X) without rewrites.

## What This Phase Delivers

1. **Automation Builder UI** — Visual interface to create, edit, enable/disable automations
2. **Trigger: Comment Keyword** — User defines one or more trigger keywords/phrases per automation
3. **Actions:**
   - Reply to the comment (public reply)
   - Send a DM to the commenter (private message)
   - Both actions can be combined in sequence
4. **Randomized Responses** — User can define a list of possible responses; system picks one randomly per trigger to appear more human
5. **Automation Rules** — Per-automation settings: active/inactive, target account, target posts (all or specific)
6. **Execution Engine** — Webhook-driven event processor that matches comments against active automations and executes actions via platform APIs
7. **Activity Log** — History of automation executions (triggered, succeeded, failed) with timestamps

## What This Phase Does NOT Deliver

- Other trigger types (DM received, story reply, mention, new follower) — future phase
- Other platforms beyond Instagram — future phase (but architecture must support them)
- Conditional logic / branching flows (if/else) — future phase
- Delay/timers between actions — future phase
- Analytics dashboard for automations — future phase
- AI-generated responses — deferred to v2

</domain>

<decisions>
## Implementation Decisions (Locked)

### D-01: Trigger Type — Comment Keyword
- Each automation has one or more trigger keywords/phrases
- Keywords are case-insensitive
- Keywords can be exact match or contains-match (the agent's discretion on default)
- Multiple keywords per automation: OR logic (any keyword triggers)

### D-02: Actions — Reply + DM
- Action 1: Reply to comment (public) — uses platform's comment reply API
- Action 2: Send DM (private) — uses platform's messaging API where available
- Actions execute in sequence: reply first, then DM
- Either action can be disabled per automation (user chooses which to enable)

### D-03: Randomized Responses
- User defines a list of 1-N response messages per action
- When triggered, system randomly selects one message from the list
- If list has 1 item, always uses that item (deterministic)
- Randomization uses simple Math.random() — no seeding needed

### D-04: Multi-Platform Architecture (Instagram First)
- Database schema must support `platform` field on all automation tables
- Execution engine must be platform-agnostic with platform-specific adapters
- Instagram is the first implemented platform
- TikTok, YouTube, X adapters are stubbed/placeholder but schema-ready

### D-05: Webhook-Driven Execution
- Comment events arrive via existing webhook infrastructure (`/api/webhooks/[platform]`)
- Webhook processor routes comment events to automation engine
- Automation engine queries active automations for the target account/post
- Matches trigger keywords, executes actions, logs results

### D-06: Per-Account Scope
- Automations are scoped to a specific `socialAccounts` record (one connected account)
- User can optionally scope to specific posts (all posts vs. selected post IDs)
- Default: all posts on the connected account

### D-07: Activity Logging
- Every automation execution is logged: automationId, triggerEvent, actionResults, timestamp, status
- Logs visible in a simple list/table UI (not charts/analytics)
- Log retention: last 100 executions per automation (configurable later)

### D-08: Rate Limiting & Safety
- Max 1 automation execution per comment (prevent loops)
- Cooldown per user: same commenter cannot trigger same automation more than once per hour
- Global rate limit: max 100 automation actions per hour per account (respects platform limits)

</decisions>

## Specific Examples

### Example Automation Flow
1. User creates automation named "Promo Link Responder"
2. Trigger keywords: ["EU QUERO", "quero", "link"]
3. Actions enabled:
   - Reply to comment (enabled): messages = ["Te enviei o link na DM!", "Mandei o link na DM pra voce!", "Confere sua DM!"]
   - Send DM (enabled): messages = ["Aqui esta o link: https://exemplo.com", "Link solicitado: https://exemplo.com"]
4. Scope: Instagram account @minhaloja, all posts
5. Status: active

### Execution Flow
1. Someone comments "EU QUERO" on a post
2. Meta webhook fires → `/api/webhooks/instagram`
3. Webhook stores event, processes it
4. Automation engine finds matching active automations for @minhaloja
5. Keyword "EU QUERO" matches automation "Promo Link Responder"
6. Engine randomly picks reply message: "Mandei o link na DM pra voce!"
7. Engine calls Instagram API to post reply to comment
8. Engine randomly picks DM message: "Aqui esta o link: https://exemplo.com"
9. Engine calls Instagram API to send DM (if permissions allow)
10. Execution logged with status "success" or "failed" per action

</specifics>
