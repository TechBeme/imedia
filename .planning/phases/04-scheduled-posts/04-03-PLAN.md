<task type="auto">
  <name>Task 1: Create cron executor for scheduled posts</name>
  <files>
    src/app/api/cron/scheduled-posts/route.ts
    vercel.json
  </files>
  <action>
    1. Create `src/app/api/cron/scheduled-posts/route.ts`:
       - Export a `GET` handler (Vercel cron calls GET)
       - Verify the request is from Vercel cron (check `Authorization` header against `CRON_SECRET` env var)
       - Query `scheduled_posts` for records where:
         - `status = 'pending'`
         - `scheduledAt <= now()`
         - `platform = 'instagram'`
       - For each post:
         a. Fetch the user's Instagram social account (from `social_accounts` table)
         b. Decrypt the access token
         c. Call the existing publish function from `src/lib/instagram.ts` (or inline the logic)
            - Determine media type from mediaUrls count and file extension
            - Call `publishImage`, `publishCarousel`, `publishReel`, or `publishStory`
         d. On success:
            - Update `scheduled_posts` set `status = 'published'`, `executedAt = now()`, `externalPostId = result.id`
            - Insert into `posts` table with `status = 'published'`
         e. On failure:
            - Update `scheduled_posts` set `status = 'failed'`, `errorMessage = error.message`
       - Return JSON summary: `{ processed: N, published: N, failed: N }`
       - Log each action with console.log for debugging

    2. Update `vercel.json` to add the cron job:
       ```json
       {
         "crons": [
           {
             "path": "/api/cron/scheduled-posts",
             "schedule": "*/5 * * * *"
           }
         ]
       }
       ```
       If `vercel.json` doesn't exist, create it. If it exists, append to the `crons` array.

    3. Add `CRON_SECRET` to the environment variables documentation (`.env.example` or similar).

    **Important:** The cron must be idempotent. If a post is already being processed (edge case), skip it. Use a simple in-memory flag or rely on the status check.
  </action>
  <verify>
    <automated>grep -c "CRON_SECRET\|scheduled_posts\|status.*published\|status.*failed" src/app/api/cron/scheduled-posts/route.ts | xargs test 4 -le</automated>
  </verify>
  <done>Cron executor exists, queries pending posts, publishes via Instagram API, and updates status correctly. Vercel cron configured for every 5 minutes.</done>
</task>
