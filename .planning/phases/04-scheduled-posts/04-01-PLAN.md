<task type="auto">
  <name>Task 2: Create /api/scheduled-posts CRUD API</name>
  <files>
    src/app/api/scheduled-posts/route.ts
  </files>
  <action>
    Create `src/app/api/scheduled-posts/route.ts` with full CRUD:

    **GET** — List scheduled posts for authenticated user:
    - Query params: `status` (optional filter: pending, published, failed, cancelled), `limit` (default 50), `offset` (default 0)
    - Returns: array of scheduled posts ordered by `scheduledAt` ASC (soonest first)
    - Use `getSession()` from `@/lib/session` for auth
    - Use Drizzle to query `scheduled_posts` table where `userId = session.user.id`

    **POST** — Create a scheduled post:
    - Body: `{ platform, content, mediaUrls[], scheduledAt }`
    - Validate: `platform` is supported (currently only "instagram"), `content` non-empty, `scheduledAt` is in the future
    - Insert into `scheduled_posts` with `status: "pending"`
    - Return the created record
    - Apply rate limiting via `withRateLimit()`

    **PUT** — Update a scheduled post:
    - Body: `{ id, content?, mediaUrls?, scheduledAt? }`
    - Only allow updates if `status === "pending"`
    - Validate ownership: post.userId === session.user.id
    - Return updated record

    **DELETE** — Cancel a scheduled post:
    - Query param: `id`
    - Only allow if `status === "pending"`
    - Soft-delete by updating `status` to `"cancelled"` (do NOT hard delete — audit trail)
    - Validate ownership

    Use standardized API responses via `createSuccessResponse` / `createErrorResponse` from `@/lib/api-response`.
    Use `withRateLimit()` from `@/lib/rate-limit` on all methods.
  </action>
  <verify>
    <automated>curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/api/scheduled-posts || echo "API file exists and exports handlers"</automated>
  </verify>
  <done>API endpoint exists with GET, POST, PUT, DELETE. All methods validate auth, ownership, and status constraints.</done>
</task>
