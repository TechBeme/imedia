         - Scheduled: skeleton cards matching the list item layout
         - History: skeleton table rows (4-5 rows) matching the table structure

         - Optional CTA button if applicable (e.g., "Create post" for scheduled)
       - Use the existing `glass-card` class

       - Add `isError` state and `retry()` function that re-triggers the mock fetch
       - When error, show a centered card with:
         - `AlertTriangle` icon in red/amber
         - Heading: i18n key `{page}.errorTitle`
         - Description: i18n key `{page}.errorDescription`
         - "Try again" button that calls retry()
