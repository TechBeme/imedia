## Meta Graph API Insights

Endpoint: `GET /<MEDIA_ID>/insights`
Metrics available:
- `impressions` — Total times the media was seen
- `reach` — Unique accounts that saw the media
- `engagement` — Total interactions (likes, comments, saves, shares)
- `saved` — Number of saves
- `shares` — Number of shares
- `likes` — Number of likes
- `comments` — Number of comments

Response format:
```json
{
  "data": [
    { "name": "impressions", "values": [{ "value": 1234 }] },
    { "name": "reach", "values": [{ "value": 567 }] }
  ]
}
```

## Meta Graph API Comments

List comments: `GET /<MEDIA_ID>/comments?fields=id,text,username,timestamp`
Reply to comment: `POST /<COMMENT_ID>/replies?message={text}`
Hide/delete comment: `DELETE /<COMMENT_ID>`

       - Calls graph.facebook.com/v22.0/{mediaId}/insights?metric=impressions,reach,engagement,saved,shares
       - Returns structured response: { impressions, reach, engagement, saves, shares }
       - If media not found or no insights available, return empty metrics
    
    2. Create src/app/api/instagram/comments/route.ts:
       - GET /api/instagram/comments?mediaId={id} — list comments
         - Calls graph.facebook.com/v22.0/{mediaId}/comments?fields=id,text,username,timestamp
         - Returns { comments: Array<{ id, text, username, timestamp }> }
       - POST /api/instagram/comments — reply to a comment
         - Body: { commentId: string, message: string }
         - Calls POST graph.facebook.com/v22.0/{commentId}/replies with message
         - Returns { success: true, replyId: string }
       - DELETE /api/instagram/comments?commentId={id} — hide/delete comment
         - Calls DELETE graph.facebook.com/v22.0/{commentId}
         - Returns { success: true }
       - All methods protected by auth and rate limiting
       - Proper error handling for each operation
