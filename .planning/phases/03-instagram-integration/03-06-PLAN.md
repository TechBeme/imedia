<objective>
Implement Instagram publishing rate limit enforcement (100 posts per 24-hour rolling window). Show users their current usage and block publishing when the limit is reached.

Purpose: Instagram enforces a hard limit of 100 API-published posts per 24 hours. Exceeding this results in API errors. This plan prevents users from hitting the limit unexpectedly.

Output: Rate limit tracking, user-facing warnings, and publish blocking when limit reached.
</objective>

## Meta Graph API Rate Limit Endpoint

`GET /<IG_ID>/content_publishing_limit`
Response:
```json
{
  "config": {
    "quota_total": 100,
    "quota_duration": 86400
  },
  "quota_usage": 42
}
```

This gives the current usage within the 24-hour window.

## Fallback: Local Tracking

If the Meta API endpoint fails or returns no data, track locally:
- Query posts table for Instagram posts published in last 24 hours
- Count: `SELECT COUNT(*) FROM posts WHERE platforms @> ARRAY['instagram'] AND status = 'published' AND publishedAt > NOW() - INTERVAL '24 hours'`
