## Current OAuth Flow (env var based)

The current auth endpoint reads:
- process.env.INSTAGRAM_APP_ID
- process.env.INSTAGRAM_APP_SECRET
- process.env.INSTAGRAM_REDIRECT_URI

The callback endpoint exchanges code via api.instagram.com/oauth/access_token, then gets long-lived token via graph.instagram.com/access_token. It then tries graph.facebook.com for Business accounts, falls back to graph.instagram.com.

## Token Refresh (Instagram Login path)

Instagram Login tokens can be refreshed via:
```
GET https://graph.instagram.com/refresh_access_token?grant_type=ig_refresh_token&access_token={long-lived-token}
```

Response: { access_token, expires_in }

## Token Revocation

No direct revocation endpoint for Instagram Login. On disconnect, delete from DB and mark inactive.
